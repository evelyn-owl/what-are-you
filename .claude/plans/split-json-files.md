# План: Розбити великі JSON файли для зручного редагування

## Проблема
5 JSON файлів по ~4000 рядків. Щоб змінити одне питання або одного героя — треба читати весь файл, витрачаючи багато токенів.

## Рішення: Розбити на маленькі файли за типом + мовою

### Нова структура (source)
```
data/
  animals/
    results.json      (~500 рядків, UA: герої з emoji, name, desc, traits)
    questions.json     (~250 рядків, UA: тексти питань)
    scoring.json       (~150 рядків, логіка: option_id → [hero_ids])
    meta.json          (~10 рядків, id, questions_count)
    i18n/
      en.json          (~200 рядків, плоскі ключі {"lion.name":"Lion", "q1.text":"..."})
      es.json
      de.json
  hp/
    results.json
    questions.json
    scoring.json
    meta.json
    i18n/en.json, es.json, de.json
  ...5 квізів
```

### Що це дає
| Задача | Зараз (рядків) | Після |
|--------|---------------|-------|
| Змінити героя UA | ~4000 | ~500 (results.json) |
| Змінити переклад DE | ~4000 | ~200 (de.json) |
| Додати питання | ~4000 | ~250 (questions.json) |
| Змінити логіку скорінгу | ~4000 | ~150 (scoring.json) |
| Додати нову мову | модифікувати всі i18n | створити 1 файл |

### Як працює в браузері
`loadQuizData()` в index.html:
1. Завантажує всі частини паралельно: meta + results + questions + scoring
2. Збирає в один об'єкт (той самий формат що зараз)
3. Накладає переклад поточної мови з i18n/{lang}.json
4. Інші мови можна підвантажити в фоні

### Кроки реалізації
1. `migrate.js` — розбиває поточні data/*.json на нову структуру
2. Оновити `loadQuizData()` в index.html
3. Оновити `toggleLang()` для підвантаження перекладів
4. Оновити `list-content.js` для нових шляхів
5. Видалити старі data/*.json
6. Перевірити що все працює

### Майбутні покращення (окремо, не зараз)
- **Google Sheets** як джерело: редагуєш таблицю з телефону → GitHub Action автоматично генерує JSON → деплой без ноутбука
- **Більше варіантів відповідей**: 8 в JSON, показувати 4 рандомних
